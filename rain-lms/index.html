Certification Structure
Certification is a continuous state — not a periodic test. The health score reflects active standing at all times and feeds all triggering events automatically.
Health score composition
Weighted composite · Live via WorkRamp Analytics Studio + Zendesk API
QA rolling average (Zendesk AutoQA)
70%
Microlearning assessment scores
20%
Module completion rate
10%
Active certification threshold
Health score must remain ≥ 80% for certification to stay active. Below 80% triggers the remediation loop automatically.
Triggering events
Events that activate automatic LMS actions — no manual intervention required
Score below threshold
Auto-assigns targeted remediation module based on the specific failing scorecard criterion. Coaching triggered if score doesn't recover.
QA fatal error
Immediate module assigned. Queue suspended until passed. Team lead auto-notified in LMS on both assignment and completion.
Product or policy change
Required update module issued with a 5-business-day window. Non-completion flags agent as non-compliant and suspends certification.
Tier advancement eligibility
LMS surfaces readiness automatically when time and QA gates are met. No manual audit or manager review needed to flag.
Assessment design principles
All performance assessments are branching simulations — not knowledge tests. Built in Articulate Storyline or natively in LMS. Path is scored, not just the final answer.
Branching scenarios
Agent decisions branch to new situations. The path taken is scored — mirrors how real interactions unfold and where judgment actually matters.
Auto-scored
No human review required. Scales to any cohort size. Attempt limits and wait periods enforced automatically by the LMS.
Scenario bank
No two agents receive identical assessments. The library expands with new contact types and edge cases as Rain scales. Resistant to answer-sharing.
Guardrails
Defined wait period between attempts and a max attempt count. Once exceeded, a remediation path is auto-triggered before the agent can retry.

QA-driven content loop
Runs continuously without manual intervention — every performance dip becomes a system input
AUTO
1
QA drop detected
AutoQA or manual score falls below threshold on a specific scorecard criterion — for one agent or as a pattern across the team.
2
Gap diagnosed
The failing criterion maps to a content category — escalation writing, KYB intake, settlement triage, partner communication, etc.
3
Content assigned or built
Existing reusable inventory is checked first. If no match, a targeted module is developed and added to the library for future use.
4
Targeted delivery
Module assigned to the specific agent or group where the gap exists. Not broadcast to the entire team — targeted, not scattered.
5
QA rechecked
Next QA cycle evaluates whether the criterion score recovers. Content effectiveness report records the delta for every module.
6
Loop continues
If score holds, monitoring continues. If content shows less than 3pp improvement, it is flagged for redesign in the effectiveness report — not recycled.
